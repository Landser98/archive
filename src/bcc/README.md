# BCC (Bank CenterCredit) - Batch Parser

## Описание

Batch-парсер для обработки выписок BCC (Bank CenterCredit). Парсит PDF файлы, валидирует данные и вычисляет доходы ИП.

## Использование

```bash
python src/bcc/batch_parse.py <path> [опции]
```

### Параметры

- `path` - Путь к PDF файлу или директории с PDF файлами (обязательный)
- `--out-dir` - Директория для выходных CSV файлов (по умолчанию: `<base_dir>_out`)
- `--jsonl-dir` - Директория для JSONL файлов (по умолчанию: `<base_dir>/converted_jsons`)
- `--pdf-meta-dir` - Директория для JSON метаданных (по умолчанию: `<base_dir>/pdf_meta`)
- `--jsonl-suffix` - Суффикс для JSONL файлов (по умолчанию: `_pages.jsonl`)
- `--months-back` - Количество месяцев для расчета доходов ИП (по умолчанию: 12)
- `--no-verbose` - Отключить подробный вывод

### Примеры

```bash
# Обработать один файл
python src/bcc/batch_parse.py data/bcc/statement.pdf

# Обработать все PDF в директории
python src/bcc/batch_parse.py data/bcc/

# Указать выходную директорию
python src/bcc/batch_parse.py data/bcc/ --out-dir results/
```

## Input (Входные данные)

### PDF файлы
- **Формат**: PDF выписки BCC (Bank CenterCredit)
- **Расположение**: `data/bcc/*.pdf`
- **Можно указать**: один файл или директорию с файлами

## Промежуточные файлы

### JSONL страниц
- **Путь**: `data/bcc/converted_jsons/<stem>_pages.jsonl`
- **Содержит**: постраничный дамп PDF в формате pdfplumber
- **Создается автоматически** при первом запуске

### JSON метаданные
- **Путь**: `data/bcc/pdf_meta/<stem>.json`
- **Содержит**: метаданные PDF (версия, создатель, даты, структура)
- **Создается автоматически** при первом запуске

## Output (Выходные данные)

Все файлы сохраняются в директории `data/bcc_out/` (или указанной через `--out-dir`):

### 1. `<stem>_header.csv`
Заголовок выписки
- Период выписки
- Информация о счете
- Начальный и конечный остаток

### 2. `<stem>_transactions.csv`
Транзакции
- Күні / Дата (дата операции)
- Дебет / Дебет
- Кредит / Кредит
- ТМК /КНП (код назначения платежа)
- Төлемнің мақсаты / Назначение платежа
- Корреспондент / Корреспондент

### 3. `<stem>_footer.csv`
Итоговая информация
- Итоговые обороты (дебет/кредит)
- Конечный остаток

### 4. `<stem>_meta.csv`
Метаданные обработки
- `bank` - Название банка (BCC)
- `pdf_file` - Имя исходного PDF файла
- `jsonl_file` - Имя JSONL файла
- `flags` - Флаги валидации (разделенные точкой с запятой)
- `debug_info` - Отладочная информация в формате JSON

### 5. `<stem>_tx_ip_enriched.csv`
Транзакции с флагами ИП
- Все колонки из `<stem>_transactions.csv`
- Дополнительные флаги для идентификации доходов ИП
- Пометки об исключениях из расчета дохода

### 6. `<stem>_ip_income_monthly.csv`
Доходы ИП по месяцам
- Месяц
- Сумма дохода за месяц
- Количество транзакций

### 7. `<stem>_income_summary.csv`
Сводка по доходам ИП
- `total_income` - Общая сумма доходов
- `total_income_adjusted` - Скорректированная сумма (с учетом исключений)
- Дополнительная статистика

## Особенности

- Использует pdfplumber для извлечения данных из PDF
- Требует JSONL файлы для парсинга (создаются автоматически)
- Выполняет числовую валидацию (если схема BCC зарегистрирована)
- Валидирует метаданные PDF (даты создания, создатель)
- Поддерживает двуязычные колонки (казахский/русский)

## Формат дат

- Формат дат в выписках: `%d.%m.%Y` (дд.мм.гггг)

## Кодировка

Все CSV файлы сохраняются с кодировкой `utf-8-sig` (UTF-8 с BOM) для корректного отображения в Excel.
