# Eurasian Bank - Batch Parser

## Описание

Batch-парсер для обработки выписок Eurasian Bank. Парсит PDF файлы, валидирует данные и вычисляет доходы ИП.

## Использование

```bash
python src/eurasian_bank/batch_parse.py <path> [опции]
```

### Параметры

- `path` - Путь к PDF файлу или директории с PDF файлами (обязательный)
- `--pattern` - Шаблон поиска файлов при указании директории (по умолчанию: `*.pdf`)
- `--out-dir` - Директория для выходных CSV файлов (по умолчанию: `<base_dir>_out`)
- `--pdf-meta-dir` - Директория для JSON метаданных (по умолчанию: `<base_dir>/pdf_meta`)
- `--pages` - Спецификация страниц для Camelot (по умолчанию: `1-end`)
- `--flavor` - Метод извлечения таблиц Camelot: `lattice` или `stream` (по умолчанию: `lattice`)
- `--months-back` - Количество месяцев для расчета доходов ИП (по умолчанию: 12)
- `--no-verbose` - Отключить подробный вывод

### Примеры

```bash
# Обработать один файл
python src/eurasian_bank/batch_parse.py data/eurasian_bank/statement.pdf

# Обработать все PDF в директории
python src/eurasian_bank/batch_parse.py data/eurasian_bank/

# Использовать метод stream для извлечения таблиц
python src/eurasian_bank/batch_parse.py data/eurasian_bank/ --flavor stream
```

## Input (Входные данные)

### PDF файлы
- **Формат**: PDF выписки Eurasian Bank
- **Расположение**: `data/eurasian_bank/*.pdf`
- **Можно указать**: один файл или директорию с файлами

## Промежуточные файлы

### JSON метаданные
- **Путь**: `data/eurasian_bank/pdf_meta/<stem>.json`
- **Содержит**: метаданные PDF (версия, создатель, даты, структура страниц)
- **Создается автоматически** при первом запуске

## Output (Выходные данные)

Все файлы сохраняются в директории `data/eurasian_bank_out/` (или указанной через `--out-dir`):

### 1. `<stem>_header.csv`
Заголовок выписки
- Период выписки (`period_start`, `period_end`)
- Информация о счете
- Начальный и конечный остаток (`opening_balance`, `closing_balance`)
- Обороты (`credit_turnover`, `debit_turnover`)

### 2. `<stem>_tx.csv`
Транзакции
- Дата проводки
- Дебет
- Кредит
- КНП (код назначения платежа)
- Назначение платежа
- Наименование Бенефициара/Отправителя

### 3. `<stem>_footer.csv`
Итоговая информация
- Итоговые обороты (`turnover_credit`, `turnover_debit`)
- Конечный остаток (`final_balance`)

### 4. `<stem>_meta.csv`
Метаданные обработки
- `bank` - Название банка (EURASIAN)
- `pdf_file` - Имя исходного PDF файла
- `flags` - Флаги валидации (разделенные точкой с запятой)
- `debug_info` - Отладочная информация в формате JSON

### 5. `<stem>_tx_ip.csv`
Транзакции с флагами ИП
- Все колонки из `<stem>_tx.csv`
- Дополнительные флаги для идентификации доходов ИП
- Пометки об исключениях из расчета дохода

### 6. `<stem>_ip_income_monthly.csv`
Доходы ИП по месяцам
- Месяц
- Сумма дохода за месяц
- Количество транзакций

### 7. `<stem>_income_summary.csv`
Сводка по доходам ИП
- `total_income` - Общая сумма доходов
- `total_income_adjusted` - Скорректированная сумма (с учетом исключений)
- Дополнительная статистика

## Особенности

- Использует Camelot для извлечения таблиц из PDF
- Поддерживает два метода извлечения: `lattice` (по умолчанию) и `stream`
- Выполняет числовую валидацию (если схема EURASIAN зарегистрирована)
- Валидирует метаданные PDF (даты создания, создатель)
- Автоматически дополняет header данными из footer при необходимости

## Формат дат

- Формат дат в выписках: `%d.%m.%Y` (дд.мм.гггг)

## Кодировка

Все CSV файлы сохраняются с кодировкой `utf-8-sig` (UTF-8 с BOM) для корректного отображения в Excel.
