# Freedom Bank - Batch Parser

## Описание

Batch-парсер для обработки выписок Freedom Bank. Парсит PDF файлы, валидирует данные и вычисляет доходы ИП.

## Использование

```bash
python src/freedom_bank/batch_parse.py <path> [опции]
```

### Параметры

- `path` - Путь к PDF файлу или директории с PDF файлами (обязательный)
- `--pattern` - Шаблон поиска файлов при указании директории (по умолчанию: `*.pdf`)
- `--out-dir` - Директория для выходных CSV файлов (по умолчанию: `<base_dir>_out`)
- `--pdf-meta-dir` - Директория для JSON метаданных (по умолчанию: `<base_dir>/pdf_meta`)
- `--months-back` - Количество месяцев для расчета доходов ИП (по умолчанию: 12)
- `--no-verbose` - Отключить подробный вывод

### Примеры

```bash
# Обработать один файл
python src/freedom_bank/batch_parse.py data/freedom/statement.pdf

# Обработать все PDF в директории
python src/freedom_bank/batch_parse.py data/freedom/

# Указать выходную директорию
python src/freedom_bank/batch_parse.py data/freedom/ --out-dir results/
```

## Input (Входные данные)

### PDF файлы
- **Формат**: PDF выписки Freedom Bank
- **Расположение**: `data/freedom/*.pdf`
- **Можно указать**: один файл или директорию с файлами

## Промежуточные файлы

### JSON метаданные
- **Путь**: `data/freedom/pdf_meta/<stem>.json`
- **Содержит**: метаданные PDF (версия, создатель, даты, структура страниц)
- **Создается автоматически** при первом запуске

## Output (Выходные данные)

Все файлы сохраняются в директории `data/freedom_out/` (или указанной через `--out-dir`):

### 1. `<stem>_header.csv`
Заголовок выписки
- Период выписки (`period_start`, `period_end`)
- Информация о счете
- Начальный и конечный остаток

### 2. `<stem>_tx.csv`
Транзакции
- Дата (дата операции)
- Дебет
- Кредит
- КНП (код назначения платежа, отсутствует в выписке - заполняется пустыми значениями)
- Назначение платежа
- Корреспондент

### 3. `<stem>_footer.csv`
Итоговая информация
- Итоговые обороты (`credit_total`, `debit_total`)
- Конечный остаток

### 4. `<stem>_meta.csv`
Метаданные обработки
- `bank` - Название банка (FREEDOM)
- `pdf_file` - Имя исходного PDF файла
- `flags` - Флаги валидации (разделенные точкой с запятой)
- `debug_info` - Отладочная информация в формате JSON

### 5. `<stem>_tx_ip.csv`
Транзакции с флагами ИП
- Все колонки из `<stem>_tx.csv`
- Дополнительные флаги для идентификации доходов ИП
- Пометки об исключениях из расчета дохода

### 6. `<stem>_ip_income_monthly.csv`
Доходы ИП по месяцам
- Месяц
- Сумма дохода за месяц
- Количество транзакций

### 7. `<stem>_income_summary.csv`
Сводка по доходам ИП
- `total_income` - Общая сумма доходов
- `total_income_adjusted` - Скорректированная сумма (с учетом исключений)
- Дополнительная статистика

## Особенности

- Использует pdfplumber для извлечения данных из PDF
- Выполняет числовую валидацию (если схема FREEDOM зарегистрирована)
- Валидирует метаданные PDF (даты создания, создатель)
- **КНП отсутствует в выписках Freedom Bank** - колонка создается с пустыми значениями
- Использует специальный формат дат: `%d.%m.%y` (дд.мм.гг)

## Формат дат

- Формат дат в выписках: `%d.%m.%y` (дд.мм.гг) - **отличается от других банков**

## Кодировка

Все CSV файлы сохраняются с кодировкой `utf-8-sig` (UTF-8 с BOM) для корректного отображения в Excel.
